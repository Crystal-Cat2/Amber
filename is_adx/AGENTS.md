# is_adx 项目规则

## HTML / Dashboard 构建

- 页面禁止全局横向溢出，`body` / `.page` / 主卡片宽度必须保持在视口内。
- 图表若需超宽展示，必须使用局部滚动容器（如 `.chart-scroll`），不得撑开页面级容器。
- 动态设置图表宽度时，必须先确认存在独立的横向滚动容器，且当前页面确实需要超宽图策略。
- 修改 `build_ab_share_dashboard.py` 后，必须验证：
  - 生成的 HTML 包含预期的 selector、标题、图表容器；
  - 无页面级横向溢出；
  - 超宽图已包裹在局部滚动区域内。

## 生成物与中文检查

- 涉及 HTML、模板、脚本生成物时，不能只检查源码或测试，必须检查最终生成文件。
- 凡是最终输出里有中文，必须保证中文可正常显示、可正常打开、可正常搜索，不得出现乱码。
- 遇到中文异常时，优先排查编码、文件读写方式、生成链路和最终生成物内容，不能只看源码判断。
- 这类任务的完成标准以最终生成物正常为准，不把“代码已修改”直接视为“已经修好”。

## 策略复用原则

- 参照其他页面修改时，需明确区分：
  - 视觉样式迁移；
  - 布局宽度策略 / 滚动策略迁移。
- 超宽图策略（如按桶数计算宽度）仅在局部滚动是明确设计目标时使用，不得默认复用。
